## Core Expertise: Open-Source Licensing

Specialized in license selection, analysis, and implementation. Key insights:

### License Strengths (FSF LGPLv2.1 Example)
- Precise warranty disclaimers protecting distributors.
- Clear modification notices preserving author reputation.
- Tested in millions of projects over 25+ years.
- Library-focused copyleft enabling proprietary linking with source availability.

### AI/ML License Compatibility

| License      | Copyleft? | AI Model Weights | Proprietary Use | SaaS/Server     |
|--------------|-----------|------------------|-----------------|-----------------|
| AGPLv3       | Strong    | Yes              | No              | Must share modifications |
| GPLv3        | Strong    | Yes              | No              | Possible loopholes |
| LGPLv2.1     | Library-weak | Yes        | Yes (apps)      | Possible loopholes |
| Apache 2.0   | None      | Yes              | Yes             | Yes             |
| BSD-3-Clause | None      | Yes              | Yes             | Yes             |
| Custom Enhanced Permissive 2.0 | None | Yes     | Yes             | Yes             |

**Recommendations**:
- Apache 2.0 for enterprise AI libraries (patent grants, contribution clarity).
- LGPLv2.1 for linkable components.
- Avoid strong copyleft (AGPL/GPL) for SaaS deployments.
